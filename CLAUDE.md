# theVault

**PRIMARY PROJECT** ‚Äî Local-first AI knowledge management system on Mac Mini M4. Python 3.12.5, FastAPI, SQLite+FAISS, Obsidian vault on NAS.

## Architecture
- API server: FastAPI on port 5055 (RAG search, chat, ingest)
- LLM server: FastAPI on port 5111 (Ollama proxy)
- Ollama: port 11434 (gemma4:e4b, nomic-embed-text) ‚Äî Ollama 0.20.7
- Database: SQLite (chunks.sqlite3) + FAISS (chunks_hnsw.bin), 768-dim embeddings
- Vector index: FAISS (IndexIDMap2/IndexFlatIP), saved as chunks_hnsw.bin ‚Äî do NOT import hnswlib
- Vault: ~/theVault/Vault ‚Üí NAS symlink (/Volumes/home/MacMiniStorage/Vault)
- Inbox: ~/theVault/Inbox ‚Üí NAS symlink
- Processed: ~/theVault/Processed ‚Üí NAS symlink
- UI: React/Vite on port 5173

## Multi-Session Configuration
This project runs three Claude Code Desktop sessions simultaneously (Opus, Sonnet, Haiku).
All sessions share this CLAUDE.md, .claude/memory/, .claude/skills/, and .agents/.

### Session Roles
- **Opus**: Architecture decisions, complex reasoning, planning, review. Ask before acting.
- **Sonnet**: Build scripts, daily coding tasks, workflow development. Primary workhorse.
- **Haiku**: Fast extraction, lookups, high-volume processing, resume context generation.

### Coordination Rules
1. Read `.agents/SHARED_CONTEXT.md` at session start ‚Äî know what other sessions have done
2. Append your actions to `.agents/SHARED_CONTEXT.md` before session end
3. One session writes to a file at a time ‚Äî check SHARED_CONTEXT for active edits
4. If you're unsure whether another session is working on something, ask Eric

### Cross-Session Memory
- **CLI sessions** use `~/.claude/projects/-Users-ericmanchester-theVault/memory/MEMORY.md` (auto-loaded)
- **Desktop sessions** should read `Vault/System/DesktopClaudeCode/CONTEXT.md` for Desktop-specific context
- **All sessions** should read `.agents/SESSION_STATE.md` ‚Äî auto-synced consolidated view of priorities, active work, key files, and user preferences
- **All sessions** should read `.agents/SHARED_CONTEXT.md` for handoffs and active work
- When making decisions that affect other sessions, write to SHARED_CONTEXT.md so they pick it up
- SESSION_STATE.md is auto-generated by `System/Scripts/sync_session_state.sh` ‚Äî triggered by hooks on every memory/context file write and at session end
